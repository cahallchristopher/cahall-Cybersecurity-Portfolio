### Explanation of Changes

**Detection, Triage, Investigation, Documentation, and Escalation:** 
- These terms align with the SOC Analyst's role in monitoring and responding to security events, providing a clear link to the job description.
- Each term is supported by lab work that involved detecting simulated threats, performing initial triage based on symptoms, conducting detailed investigations using network traffic analysis tools like PCAPs, documenting findings thoroughly, and escalating incidents as necessary.

**Network Traffic Analysis:**
- Emphasizes experience in analyzing network traffic (TCP sessions, DNS requests, HTTP transactions) to identify security threats.
- This is a key skill for SOC Analysts who need to understand how data flows across networks.

**DNS Requests, HTTP Transactions, TCP Sessions:**
- Specific technical skills that demonstrate familiarity with common protocols used in cybersecurity.
- These terms align with the technical expectations of analyzing network traffic and understanding endpoint behavior.

**SIEM, EDR/XDR:**
- While not explicitly mentioned in lab work, these tools are often used by SOC Analysts to detect and respond to security events. Mentioning them shows familiarity with industry-standard tools and processes.
- "SIEM" (Security Information and Event Management) and "EDR/XDR" (Endpoint Detection and Response/Extended Detection and Response) are included as they represent the types of systems an analyst might interact with in a production environment.

**Endpoint Monitoring:**
- This term highlights experience in monitoring endpoints for security threats, which is crucial for both detecting and responding to incidents.

Overall, these changes ensure that the resume aligns closely with the target job description by emphasizing relevant skills and experiences in SOC-style investigative work.
